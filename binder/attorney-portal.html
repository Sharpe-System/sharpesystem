Attorney Portal (Preview)

This is a workflow preview. No authentication. No client data. No messaging. The intent is to show how an attorney-facing view would stay clean, bounded, and non-advisory.

What this page represents

  • Read-only by default (concept)
  • Structured intake summary instead of narrative sprawl
  • Document checklist + timeline slots
  • Scope clarity to reduce confusion and liability ambiguity

Mock “matter view” tiles

1) Snapshot

One paragraph, structured. No argument. No conclusion.

Case type • Next date • Primary goal • Immediate risks

2) Timeline anchors

Dates + events only. No adjectives.

  • YYYY-MM-DD — event label
  • YYYY-MM-DD — filing / service / hearing
  • YYYY-MM-DD — communication log reference

3) Document completeness

What exists, what’s missing, and what the client claims exists but hasn’t provided.

  • Provided: X
  • Missing: Y
  • Unknown: Z
